Operating Systems Terminology
Term Definition
operating system The software that controls the overall operation fo the computer including file management, the user interface, and the program execution environment
job queue A list of tasks to be carried out in turn by an operating system
FIFO The objects are removed from the queue in the order in which they arrived
interactive processing The process of interacting directly with a program and/or data for immediately modifying, retriecing and/or displaying information
time-sharing The operation of a computer system by several users for different operations at the same time
multiprogramming The running of two or more programs or sequences of instructions simultaneously by a computer iwht more than one central processor
application software A type of computer program that performs a specific personal, educational, and business function
utility software Consists of software units that extend the capabilities of the operating system
graphical user interface (GUI) A visual way of interacting with a computer using items such as windows, icons, and menus, used by most modern operating systems
shell A UNIX term for the interactive user interface with an operating system
kernel A core component of an operating system and serves as the main interface between the computer's physical hardware and the processes running on it
device driver A set of files that tells a piece of hardware how to function by communicating with computer's operating system
file manager A system software responsible for the creation, deletion, modification of the files and managing their access, security and the resources used by them
memory manager The process of controlling and coordinating a computer's main memory
virtual memory A common technique used in a computer's operating system
boot strapping To boot or to load a program into a computer using a much smaller initial program to load in the desired program
ROM A non-volatile memory type
deadlock A condition that may happen in a system composed of multiple processes that can access shared resources
spooling A process in which data is temporarily held to be used and executed by a device, program or the system
super user (or administrator) A user that is authorized (and therefore, trusted) to perform security-relevant functions that ordinary users are not authorized to perform
multi-factor authentication A multi-step account login process that requires users to enter more information than just a password
