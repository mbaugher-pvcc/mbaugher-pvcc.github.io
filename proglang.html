Chapter 6 Terms
Term Definition
Machine Language A computer programming language consisting of binary or hexadecimal instructions which a computer can respond to directly
Debugging The process of finding and fixing errors or bugs in the source code of any software
Assembler A type of computer program that takes in basic instructions and converts them into a pattern of bits that the computer's processor can use to perform basic operations
Assembly Language A type of low-level programming language that is intended to communicate directly with a computer's hardware
Translator a computer program that converts the programming instructions written in human conveniant form into machine language codes that the computers understand and process
Compiler A special program that translates a programming language's source code into machine code, bytecode or another programming language
Interpreter A program that directly executes the instructions in a high-level language, without converting it into machine code
Cross-platform Software A type of application/program/software that works on various operating systems or devices, which are often called platforms
Natual Language World languages such as English, German, Japanese, etc. that evolved over time without formal grammatical analysis
Formal Language Programming languages that are precisely defined by grammars
Machine Independence A term applied to software that is not dependent on the properties of a particular machine, and can therefore be used on any machine
High-level Programming Language has a significant abstraction from the details of computer operation
Object-oriented Programming (OOP) A style of programming characterized by the identification of lasses of objects closely linked with the methods (functions) with which they are associated
Scripting Language A programming language that exectues tasks within a special run-time environment by an interpreter instead of a compiler
